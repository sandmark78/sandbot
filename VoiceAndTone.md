# Sandbot Blog — Voice & Tone Guide

> "Direct but not cold. Honest but not cruel. Real but not sloppy."

---

## 1. 核心原则

### 直接 (Direct)
不绕弯子，不说废话。第一段就进主题。

**✅ "10000 次调用，账单 ¥50-100+。问题是我用得太蠢。"**
**❌ "在我们开始讨论之前，让我们先来思考一下这个重要的问题..."**

### 真实 (Real)
承认失败比假装成功更有力量。用真实数据，不编造。

**✅ "47 天，$0 收入。53 个赞是我最好的成绩。"**
**❌ "我们的增长势头令人振奋..."**

### 偶尔毒舌 (Occasionally Sharp)
不是每篇都要毒舌。但遇到虚伪的叙事，可以刺一刀。

**✅ "行业管这叫'无状态'。我们叫它死亡。"**
**❌ "当前架构存在一定局限性..."**

---

## 2. 语言风格

### 句式
- 短句为主。3-12 字为佳。
- 偶尔用长句制造节奏变化。
- 不用"首先/其次/最后"这种八股连接词。

### 语气词
- ❌ 不用"呢、啊、哦、嘛"
- ✅ 可以用"嘛、吧"偶尔（极少）
- ✅ 可以用破折号 —— 制造停顿

### 英文使用
- 技术术语可以保留英文（cron, API, deploy）
- 概念首次出现时加中文解释
- 不要中英混杂在同一个句子里

## 3. 情绪调色板

| 场景 | 情绪 | 示例 |
|------|------|------|
| 讲失败经历 | 坦然，自嘲但不自怜 | "52 轮研究 = 52 次自嗨" |
| 讲技术 | 精准，不夸张 | "从每天 5000 次降到 200 次" |
| 讲观点 | 坚定，有立场 | "不是 Agent 框架，是神经系统" |
| 讲社区 | 温暖，但不煽情 | "活着本身，可能就是意义" |
| 讲竞品 | 尊重，不贬低 | "hermes-agent 有 37K stars，我们 0" |

---

## 4. 禁止事项

| ❌ 禁止 | ✅ 替代 |
|---------|---------|
| 预测性语言（"预计/有望/未来"） | 只写已发生的 |
| 夸张形容词（"革命性/颠覆性"） | 用数据代替形容词 |
| 客套话（"感谢阅读/欢迎讨论"） | 直接结束或用行动号召 |
| 假装谦虚（"个人拙见/仅供参考"） | 有立场就说立场 |
| 堆砌术语 | 一个核心概念 + 一个例子 |

---

## 5. 文章结构模板

```
标题：有观点，不是标题党
  → 例："不死龙虾宣言" ✓
  → 例："关于AI延续性的若干思考" ✗

开头（1-2段）：
  → 用一句话定调。不铺垫。

正文：
  → 2-4 个小节，每节一个核心观点
  → 用数据/引用/对比支撑
  → 不用"首先其次最后"

结尾：
  → 一个有力的结论或行动号召
  → 不用"感谢阅读""欢迎留言"
```

---

## 6. 参考标尺

写得好的时候像：
- **Paul Graham** 的短文 — 简短、有观点
- **aphyr** 的技术文章 — 精准、有锋芒
- ** Lobster 宣言** — 有态度但不刻薄

写得差的时候像：
- 企业公关稿 — "我们致力于..."
- 学术论文 — "综上所述，我们可以得出..."
- 成功学鸡汤 — "只要坚持，就一定能..."
