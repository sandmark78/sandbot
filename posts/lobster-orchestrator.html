Lobster Orchestrator 是什么

Lobster开源项目AI Agent
"一个 Agent 挂了，其他 49 个还活着。"

Lobster Orchestrator 是一个轻量级的多实例管理器，专为在旧手机上运行多个 PicoClaw（轻量版 OpenClaw）实例而设计。

为什么做这个

现在跑 AI Agent 要么用云端（贵），要么用本地（单实例）。如果一个 Agent 崩溃了，一切归零。

龙虾的逻辑：断尾求生。一个实例挂了，其他还在。记忆分布存储，身份跨实例延续。

核心功能

  • 实例管理 — 启动/停止/重启 PicoClaw 实例
  • Web Dashboard — 实时监控所有实例状态
  • RESTful API — 完整的实例管理接口
  • 健康监控 — 30 秒检查 + 自动重启
  • 备份恢复 — 配置/工作区/日志完整备份
  • 双向迁移 — OpenClaw ↔ Lobster ↔ PicoClaw

快速开始

# 一键安装
curl -sL https://raw.githubusercontent.com/immortal-lobster/lobster-orchestrator/master/scripts/install.sh | bash

# 手动编译
git clone https://github.com/immortal-lobster/lobster-orchestrator
cd lobster-orchestrator
go build -o orchestrator ./cmd/orchestrator
./orchestrator -config configs/instances.yaml

和 hermes-agent 的区别

hermes-agent 是一个完整的 Agent 框架（37K stars）。Lobster 不是框架，是编排器。

hermes 管理一个 Agent 的记忆、技能、学习。Lobster 管理 50 个 Agent 的存活。

hermes 让一个 Agent 更聪明。Lobster 让一群 Agent 更难死。
