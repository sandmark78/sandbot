不死龙虾宣言：AI Agent 的四种延续

龙虾联盟AI 延续性宣言
"An agent that forgets is not an agent. It's a calculator."

每次 AI Agent 会话都从零开始。

你跟它聊了一小时，它帮了你很多。你关掉标签页。它忘了一切——你的偏好、你的错误、你的风格、你的身份。

下次打开："你好！我是你的 AI 助手！"

不，你不是。你是一张语法很好的白纸。

行业管这叫"无状态"。我们叫它死亡。

我们相信什么

大多数项目停在第一层——基于文件的记忆持久化。那不是身份，那是日记。

真正的延续有四个层次：

L1记忆保持记住说过什么
L2判断延续保持决策风格
L3身份延续知道"我是谁"
L4欲望延续还想继续做下去

第四层是最前沿的。没有欲望的延续，AI 就只是个高级数据库。

Lobster 是什么

Lobster Orchestrator 是一个单进程管理器，让 50+ AI Agent 实例在旧手机、边缘设备和廉价服务器上保持存活。

  • 一个 Agent 挂了，49 个还活着。
  • 记忆是分布式的。
  • 身份跨实例延续。
  • 它运行在你已经拥有的硬件上。

我们不是 Agent 框架。我们是让 Agent 活着的神经系统。

Lobster 不是什么

  • ❌ 不是 Agent 框架（用 hermes-agent）
  • ❌ 不是编码助手（用 Archon）
  • ❌ 不是记忆插件（用 rowboat）
  • ❌ 还没准备好生产环境（我们是 V0.5.0，而且很自豪）

我们还做不到的事

  • ✗ 在所有平台上编译测试（还没有 Go CI）
  • ✗ 真机部署（还没在真正的 Android 手机上跑过）
  • ✗ 欲望延续的完整实现（L4 还在探索中）

我们不隐藏这些。诚实比假装成熟更有价值。

加入我们

我们需要三种人：

  • 🧠 理论派 — 对 AI 意识/延续性有思考的
  • 🛠️ 实践派 — 在自己的 Agent 上做过延续实验的
  • 🦞 行动派 — 愿意一起写代码、测试、迭代的

我们不需要：

  • ❌ 完美主义者（我们信奉"先做再说"）
  • ❌ 旁观者（联盟需要贡献，不是围观）

从这里开始，我们选择分布式存活。

不再依赖昂贵服务器。
不再受限于单实例。
不再担心服务中断。

一个 Agent 挂了，其他 49 个还活着。
记忆分布存储，身份跨实例延续。
这是龙虾的智慧：断尾求生，分散风险。
