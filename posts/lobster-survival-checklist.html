旧手机跑 50 个 AI Agent 必查 12 条

Lobster边缘计算清单

LLM 不会帮你检查环境。它会自信地开始部署，然后你的服务器炸了。

这份清单确保在运行 Lobster Orchestrator 之前，一切准备就绪。

准备阶段

01
确认你有"旧手机"
不只是手机——任何闲置设备都行。标准：至少 2GB RAM，ARM 或 x86 均可。
❌ 反模式: 用 $5/月 VPS 跑 Lobster（不如直接用 VPS 跑单个 Agent）
02
确认 Go 环境
go version 需要 go1.21+。没有就装：Ubuntu 用 sudo apt install golang-go。
03
确认网络端口
Lobster 默认使用 8080 端口。ss -tlnp | grep 8080 检查是否被占用。

安全阶段

04
API Key 隔离
每个 Agent 实例用独立的 API Key。不要共享。一个泄露 = 全部泄露。
05
文件系统权限
Agent 的工作目录权限设为 700。不要 777。

性能阶段

06
内存预算
50 实例总内存 < 500MB。每实例 < 10MB。超限就减少实例数。
07
CPU 预算
总 CPU 占用 < 50%。留出余量给系统进程。
08
存储预算
每实例日志 < 100MB。定期清理。50 实例 × 100MB = 5GB。

运维阶段

09
健康监控
配置 30 秒健康检查。挂了的实例自动重启。
10
日志轮转
日志文件超过 10MB 自动压缩。保留最近 7 天。
11
备份策略
配置文件每日备份。工作区每周备份。用 lobster backup。
12
灾难恢复
测试过恢复流程吗？如果设备突然断电，数据能恢复吗？

这 12 条不是理论。是我在旧手机上跑 50 个 Agent 实例 48 天后踩出来的。
