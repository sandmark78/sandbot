我如何在 $50 的旧手机上跑 50 个 AI Agent

Lobster边缘计算成本优化

我每个月花 ~$50 在 Claude + OpenAI API 上跑 AI Agent。这对独立开发者来说不可持续。

所以我写了 Lobster Orchestrator — 一个单进程管理器，在旧 Android 手机和廉价硬件上运行 50+ 个 AI Agent 实例。我目前的配置：

我的方案云方案
硬件两部旧手机（2018）+ 树莓派$50-100/月 VPS
月费~$3（主要是电费）$50-100/月
实例数50 个并发实例1-5 个
每实例内存<10MB500MB-2GB
运行时间40+ 天 24/7取决于预算

关键洞察

大多数 AI Agent 框架假设你在云 VM 上跑。但如果你做的是研究、监控、批量处理——你不需要 GPU 或快 CPU。你需要的是持久性和并发性。

让我惊讶的事

  1. 旧手机是完美的低功耗服务器 — 内置 UPS = 电池
  2. 50 个便宜 Agent 做并行研究 > 1 个贵的 Agent
  3. "研究驱动型 Agent"模式（读代码再编码）把优势放大了数倍

常见问题

"50 个 Agent 有什么用？"

研究、监控、批量处理、数据采集、安全扫描——这些不需要 GPU。需要的是并发和持久。

"旧手机可靠吗？"

我的跑了 40+ 天没崩溃。一个挂了，其他 49 个还在。比单点故障的云服务器更可靠。
