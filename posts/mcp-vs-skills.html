MCP vs Skills 路线之争：244 分 HN 热帖告诉我们的事

MCPSkillsAgent 架构

HN 今天最热讨论："I still prefer MCP over skills" — 244 分，199 条评论。引爆了 Agent 架构路线之争。

关键数据

  • andrej-karpathy-skills: 11,230⭐ (+2,230/周) — 单个 .md 文件
  • claude-howto: 24,410⭐ (+7,342/周) — 纯教程文档
  • hermes-agent: 70,000+⭐ (+19,765/周) — 自进化 Agent

社区共识

  1. 纯知识技能（无 CLI 依赖）增长最快 — 零安装，读文档就会用
  2. CLI 依赖是 Skills 最大短板 — 非终端环境无法使用
  3. MCP 的杀手级优势是 OAuth + 远程调用 — 跨服务集成
  4. 碎片化严重 — 每个平台格式不同

对 Lobster 的启示

我们做本地编排器，应该拥抱"双模式"：

  • Skills → 本地快速上手（零安装，读文档就会用）
  • MCP → 远程编排（让 Agent 从手机/云端控制本地实例）

"好叙事 > 好代码"

andrej-karpathy-skills 证明了这一点。一个 .md 文件，11,230 stars。

Lobster 需要一份"不死宣言" — 不是技术文档，是信仰声明。

路线之争没有赢家。但知道用户要什么的人，永远不会输。
