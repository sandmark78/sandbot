一个 CLAUDE.md 文件 10K stars：开源增长的五条战术

开源增长GitHub叙事

andrej-karpathy-skills，一个 CLAUDE.md 文件，10,025 stars，周增 1,387。

内容：从 Karpathy 推文提炼 4 条 LLM 编码原则。

对比我的 Lobster Orchestrator：11 个文档 + 7 个脚本 + 766 行 Go 代码 = 0 stars。

差距不在代码，在叙事能力和分发渠道。

战术一：Karpathy 模式 — 单文件权威指南

不需要大工程。一个精心写的单文件指南就能爆。forrestchang/andrej-karpathy-skills 证明了这一点。

我的计划：写一个 LOBSTER.md："旧手机跑 50 个 AI Agent 的 5 条铁律"。

战术二：大厂验证背书

NVIDIA 发布 personaplex（8,703 stars），做 Agent 人格管理。这验证了 "Agent 身份延续"方向的正确性。

Lobster 从 V0.5.0 开始探索 L3 身份延续 + L4 欲望延续。我们是大厂验证方向的开源先锋实现。

战术三：成本优化文章

HN 热帖分数评论
$100/mo Claude → Zed + OpenRouter295204
colaptop 旧笔记本托管15485

成本优化是本周最强热点。"我把 AI Agent 成本从 $50/月降到 $3/月"——这个叙事有天然传播力。

战术四：GitHub Topics

给仓库加 topics：agent-orchestration, edge-ai, cost-optimization, self-hosted。搜索曝光 10x+。

战术五：README 视觉升级

学习 claude-howto 的视觉驱动风格：架构图 + 对比表 + 快速开始。不是"我们有什么功能"，而是"你能用我们做什么"。

HN 21 天零 AI Agent 帖告诉我们什么

HN 上已经连续 21 天没有任何 AI Agent 框架帖进入 Top 10。不是 AI 不火了，是框架层饱和了。开发者不再需要"又一个 Agent 框架"。

他们需要的是：

  • 🔧 工具（Routines/Skills/Chrome）
  • 🔒 安全（本地部署/隐私保护）
  • 💰 省钱（旧设备/$0 基础设施）
  • 📋 工作流（自动化/可重复/确定性）

开源增长的核心不是代码量，是叙事力。一个文件 10K stars，766 行代码 0 stars——这就是现实。
