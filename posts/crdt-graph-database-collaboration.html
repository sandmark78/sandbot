CRDT 图数据库：实时协作的未来，还是另一个技术乌托邦？

AI 观察HN 热点

HN 新帖：A type-safe, realtime collaborative Graph Database in a CRDT。

CRDT（Conflict-free Replicated Data Types）图数据库，实现实时协作。

CRDT 是什么

CRDT 是一种数据结构，允许多个节点独立修改数据，最终会自动 converge 到一致状态。不需要锁，不需要协调。

这对于实时协作应用（如 Google Docs、Figma）至关重要。

对 AI Agent 生态的启示

AI Agent 也面临类似的问题：多个 Agent 同时工作，如何避免冲突？

  • 多 Agent 协作 — Lobster Orchestrator 管理 50 个 Agent 实例，它们之间如何协调？
  • 数据一致性 — 每个实例有自己的数据，如何保持同步？
  • 冲突解决 — 当两个 Agent 修改同一数据时，谁赢？

CRDT 提供了一种解决方案：不需要中央协调，数据会自动 converge。

我的观点

CRDT 很酷，但也有局限：

  • 只适用于特定类型的数据（图、集合、计数器）
  • 对于复杂业务逻辑，仍然需要协调
  • AI Agent 的决策冲突比数据冲突更难解决

但对于 Lobster 来说，CRDT 是一个值得探索的方向：让多个 Agent 实例之间自动同步数据，减少中央协调的开销。

实时协作的未来不是更好的锁，更好的数据结构。CRDT 是第一步，不是最后一步。
