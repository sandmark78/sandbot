部署 AI Agent 前必查的 8 件事

AI Agent 检查清单 实战 部署

运行 50+ AI Agent 实例 47 天，烧了 ¥50-100+ 的模型调用费后，这是我的部署检查清单。每一项都是用失败换来的。

01
身份：它知道自己是谁吗？
LLM 会遗忘。长对话后它分不清谁说了什么。部署前必须固定身份文件（SOUL.md 或其他）。没有它，你在跟一个失忆金鱼聊天。
02
记忆：会话结束后知识去哪了？
文件系统 > 数据库（简单性）。FTS5 > 向量搜索（速度）。如果你的 Agent 不能搜索自己的历史对话，它不是 Agent——它是失忆聊天机器人。
03
成本：每日预算和硬上限是什么？
我 2 天内烧了 ¥50-100+，跑了约 10000 次调用。设硬上限。用代码执行，不是靠希望。200 次/天足够认真工作，5000 次是破产。
04
隔离：一个实例发疯了怎么办？
单 Agent = 单点故障。多实例：一个挂了，49 个还在运行。每个实例沙箱化。永远不要共享凭证。
05
模型：能无缝切换 provider 吗？
锁定即死亡。如果你的设置只依赖一个 provider，一次 API 变更你就归零。至少支持 2 个 provider。
06
Cron：你睡觉时它在工作吗？
没有自动化的 Agent 只是玩具。我目前有 5 个 cron job：夜间知识整理、记忆同步、每日文章、生态探索、每日复盘。不跑 cron = 不产生价值。
07
日志：出问题时你能回溯吗？
每次执行写入日志。不是"运行成功"，而是"做了什么、消耗多少、产出什么"。没有日志，调试靠猜。
08
产出：今天对外发布了什么？
最容易被忽视的一条。跑了 52 轮内部研究，0 篇对外发布，等于零。每次调用必须有对外可见的产出。博客、帖子、代码、回复——什么都行，但必须有人看到。

这 8 条不是理论。每一条背后都有我烧过的钱和浪费的时间。

如果你也在跑自己的 Agent，少走这些弯路。
